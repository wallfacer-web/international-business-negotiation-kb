# 02 工作台总览（网页版）

这页是 `02 工作台总览.canvas` 的网页可展示版。  
工作台层负责把知识转成任务，把任务转成模板输出，把课堂活动组织成可执行流程。

## 主流程

`谈判准备 -> 信息收集 -> 对手分析 -> 策略讨论 -> 模拟演练 -> 复盘`

## 六个核心工作台

### 谈判准备

作用：明确目标、BATNA、底线、议题与角色分工。  
对应模板：[[90 模板/谈判准备单]]

入口：
- [[02 工作台/谈判准备工作台]]

### 信息收集

作用：补齐外部资料、事实依据和关键未知信息。  
对应模板：[[90 模板/信息收集卡]]

入口：
- [[02 工作台/信息收集工作台]]

### 对手分析

作用：形成对方利益、约束、权限、文化与沟通风格的工作假设。  
对应模板：[[90 模板/对手分析卡]]

入口：
- [[02 工作台/对手分析工作台]]

### 策略讨论

作用：比较多方案，判断交换条件、让步节奏和推荐路径。  
对应模板：[[90 模板/策略讨论单]]

入口：
- [[02 工作台/策略讨论工作台]]

### 模拟演练

作用：把准备和策略转成课堂角色扮演与过程观察。  
对应模板：[[90 模板/模拟演练脚本]]

入口：
- [[02 工作台/模拟演练工作台]]

### 复盘

作用：诊断问题并形成回流改进。  
对应模板：[[90 模板/谈判复盘单]]、[[90 模板/价值引导记录卡]]

入口：
- [[02 工作台/复盘工作台]]

## 核心原则

- 按阶段推进，不跳步
- 信息不足时先补信息
- 对手不清时先做画像
- 策略讨论前先明确边界
- 模拟后必须复盘

## 建议使用顺序

按标准路径推进：  
`谈判准备 -> 信息收集 -> 对手分析 -> 策略讨论 -> 模拟演练 -> 复盘`

如果中途发现信息缺口或判断错误，应回到前一工作台，而不是硬往后推。
